A blog, or weblog, is an online journal. Blogging is a simple way to share your thoughts with anyone who has internet access. In prior versions Journler included basic support for LiveJournal and MetaWeblog systems. After a great deal of discussion on the Journler forums, it has been decided to remove this support until it can be substantially improved.

If you relied heavily on Journler for blogging, you will need to find a 3rd party application to fill that void. A number of excellent options are available, among them MarsEdit and Ecto. To fascilitate 3rd party integration, Journler 2.5.4 implements the Weblog editor protocol supported by a number of these applications and also allows you to send entries to other applications or use Apple Scripts to post your entries to the web.

Setting up 3rd Party Integration

You'll find a single setting for 3rd party blogging integration in Journler's advanced preferences. Next to the Post to Weblog Using label you'll find a popup button. Click it once and select the Choose... option.

You may choose one of three kinds of files:

  1. An application that supports the weblog editor protocol, including...
    • MarsEdit
    • Ecto
    • Xjournal
    • Blapp
    • BlogThing
  2. Any other application
  3. An Apple Script

When you choose an application that supports the weblog editor protocol, Journler makes use of a behind the scenes technology and special variables to send your entry to it. When you select any other application, Journler sends a rich text version of the entry to the application. If you choose an Apple Script, the script has complete control over how the entry is presented to the 3rd party application. An Apple Script can even send your entry directly to a blog!

Tip: By default Journler sends a plain text version of the entry to applications that support the weblog editor protocol. If you would rather have Journler send an html version of the entry you may enable a hidden preference. Open up the Terminal application in your /Applications/Utilities folder and type "defaults write com.phildow.journler SendHTMLToWeblogEditor 1" without the quotes. Press return. You can disable that option by replacing the 1 with a 0. When enabled, Journler will use the XHTML/HTML and css preferences you've set for exporting to render the entry's (x)html.
